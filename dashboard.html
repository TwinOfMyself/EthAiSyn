PPT Adoption Framework
Design Rationale
P
People
No new documentation burden — .cmutel fields only
CM sees only their own patients by default
Colour-coded urgency — no training needed to triage
TL audit is automated — not a new task
P
Process
Mirrors the 5-node decision tree exactly
Node 3 triggers automatically on Friday EOD
Escalation thresholds match guideline timeframes
Closed commitments archived — full audit trail
T
Technology
Sits inside existing .cmutel — no separate login
No PHI displayed on summary view
Works on mobile — CM can update from any device
Alert sent via in-basket — no new channel
Adoption design principle: Every feature on this dashboard corresponds to a gap identified in the IMH workflow analysis. Nothing was added for aesthetics. The dashboard is the protocol made visible.
Guideline Alignment · What This Dashboard Enforces
3 Guidelines
SAMHSA 2019
24-hour follow-up as institutional responsibility. Dashboard auto-flags commitments approaching 24hr without resolution.
Joint Commission
Action list + named owner at every handoff. Every open commitment on dashboard has an owner field and status — no anonymous floating tasks.
OMH/DOH 2026
Failed attempts must be documented and visible. Every contact attempt is timestamped. Weekly audit makes broken promises countable.
Barriers to Adoption — Addressed
✗ Privacy concerns: Dashboard shows commitment status only — no clinical notes, diagnoses, or sensitive data visible at summary level.
✗ Documentation burden: No new forms. The .cmutel template gets one additional field — open commitment Y/N. Everything else is auto-populated.
✗ Workflow disruption: Dashboard mirrors the 5-node decision tree the CMs already know. No new process — existing protocol made visible.
✗ Integration costs: Designed to sit inside existing EMR in-basket system. No new infrastructure required for Phase 1.
